Project 1: Data Warehousing for Aircall Data

Project Overview

Services through phone calls facilitated by Aircall, a cloud-based telephony platform. The board tasked me with building a data warehouse to centralize, organize, and analyze team performance data.

The data warehouse provides insights into key metrics such as call duration, advisor availability, and patient engagement, enabling management to evaluate team performance and operational efficiency.

Objectives

  • Data Centralization: Integrate raw data from the Aircall API into a structured data warehouse.
  • Data Modeling: Design a scalable schema to store key metrics.
  • Performance Monitoring: Enable analysis of team effectiveness through key metrics.
  • Visualization: Provide actionable insights via interactive Tableau dashboards.

Project Workflow

Data Integration

  • Connected to the Aircall API to extract call logs, availability, and engagement data.
  • Automated data ingestion pipelines to store data in MariaDB.

Data Warehouse Design

  • Designed a star schema with fact and dimension tables.
  • Normalized data to ensure scalability and efficient querying.

ETL Pipeline Development

  • Developed an ETL process to:
    • Extract raw data from the Aircall API.
    • Transform data into the required format.
    • Load processed data into MariaDB tables.

Data Analysis

  • Queried data to compute key performance metrics:
    • Total calls per advisor.
    • Average call duration and trends.
    • Advisor availability rates.
    • Patient interaction success rates.

Dashboard Creation

  • Designed Tableau dashboards to visualize:
    • Team and individual performance.
    • Trends in call activity.
    • Metrics for patient engagement.

Challenges and Solutions

API Data Integration

Challenge: Complex, nested data structures from the Aircall API.
Solution: Flattened data structures during ETL for better compatibility.

Schema Design

Challenge: Balancing schema flexibility with query performance.
Solution: Implemented a star schema for efficiency.

Data Quality

Challenge: Inconsistent and missing data.
Solution: Added validation checks and transformation rules in ETL.

Stakeholder Reporting

Challenge: Communicating technical insights in a non-technical format.
Solution: Developed simplified Tableau dashboards with tooltips.

Technologies Used

  • Data Integration: Aircall API
  • Data Warehouse: MariaDB
  • ETL Pipeline: SQL, Python
  • Data Visualization: Tableau

Outcome

  • Centralized Data Warehouse: Built a robust data warehouse for PAT performance data.
  • Actionable Insights: Enabled tracking of advisor performance and efficiency.
  • Scalable Solution: Designed a schema capable of handling growing data volumes.
